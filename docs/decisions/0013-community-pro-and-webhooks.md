# ADR 0013: Community／ProエディションとWebhook

- 状態: Accepted
- 日付: 2026-07-14
- 現在仕様: Webhookの配送契約はADR 0014で更新し、業務payloadは含めない。基本SMTP通知、Pro実装、正式なCommunity配布は未実装

## 決定

SyncCoordinatorは、無償のCommunityエディションと商用のProエディションを提供する。

CommunityはApache License 2.0でソースコードを公開し、個人・法人を問わず商用環境および本番環境で使用できるものとする。接続システム数、同期ルール数、同期件数による機能制限は設けない。基本的な同期、競合検出と手動解決、管理画面、汎用Webhook、基本SMTPメール通知はCommunityに含める。

Proは、Communityの本番利用権を販売するものではなく、企業運用を容易にする追加機能、長期サポート版、更新提供およびサポートを提供する商用エディションとする。候補となる追加機能は、Teams等のサービス固有連携、競合の一括解決と自動解決ルール、承認フロー、失敗データの一括再実行、比較・修復、監査エクスポート、高度な通知テンプレート、条件フィルター、エスカレーションである。Pro固有の実装とライセンス発行機構は公開リポジトリから分離する。

Proライセンスはオフラインで検証できる非対称署名付きライセンスとする。公開リポジトリおよび配布物には検証用公開鍵だけを含め、署名秘密鍵、購入者情報、決済連携の秘密情報を含めない。Proライセンスの不存在、更新権の終了、検証障害を理由としてCommunityの基本同期を停止しない。買い切りライセンスを発行する場合は利用権を失効させず、更新権期限によって利用可能な新バージョンとサポート範囲を制御する。

## Webhook

業務アプリケーションへ同期結果を通知する汎用WebhookはCommunityに含める。少なくとも次のイベントを対象とする。

- 同期適用
- 削除同期
- 競合検出
- 同期失敗
- システムの一時停止と再開

Webhookイベントは同期処理から直接HTTP送信せず、Coordinator管理DBのOutboxへ永続化し、別の配送処理が送信する。配送はat-least-onceとし、受信側が`EventId`で重複排除できる契約にする。HMAC等による送信元署名、タイムアウト、段階的な再試行、配送履歴をCommunityの基本機能に含める。

通知payloadは既定でイベント種別、発生日時、同期ルール、送信元・送信先、対象識別子、操作種別、配送メッセージID等のメタデータに限定する。業務データ全体は既定では含めず、必要な場合だけ明示的に設定するか、通知を受けた業務アプリケーションが認可されたAPIから取得する。

## 理由

Communityを評価専用または小規模用途へ制限すると、A⇄C・B⇄CというSyncCoordinatorの中心的な構成を十分に検証できず、利用者と実運用実績を得にくい。無料版も本番で安全に使える品質とし、Proは同期の成立に不可欠な機能ではなく、運用負担を減らす価値で差別化する。

Webhookは業務アプリケーションとの連携に必要な基本境界であり、署名、再試行、重複排除を有料機能にすると無償版の安全性が下がる。そのため、信頼できる汎用Webhook配送をCommunityへ含め、特定サービス向けアダプターや高度な運用機能をProの候補とする。

同様に、検出した競合を安全に解決し、基本的なメールで運用者へ知らせるところまでを通常運用の一巡とみなす。Communityに個別の手動競合解決と基本SMTP通知を含め、Proは大規模運用向けの自動化、統制、サービス固有連携で差別化する。

## 帰結

- Communityの利用者は、本番・商用環境を含めて無償で基本同期を運用できる。
- Apache License 2.0に従う限り、第三者による改変、再配布、商用利用も許可される。Proはライセンス制限ではなく、非公開の追加実装とサポートによって価値を維持する必要がある。
- CommunityはProなしで単独動作し、Proは公開された拡張境界を通じて追加される。
- GitHub公開前にApache License 2.0の本文、第三者ライセンス表示、セキュリティ報告方針、秘密情報を除外する設定を整備する。
- Proの価格、購入経路、最終的な機能一覧、ライセンスファイル形式は別途決定する。
