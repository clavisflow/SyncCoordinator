# ADR 0012: ローカル管理者認証と復旧

- 状態: Accepted
- 日付: 2026-07-14

## 決定

管理画面は固定ユーザー名`admin`の単一管理者アカウントを使用する。認証はASP.NET CoreのCookie認証、パスワード保存は`PasswordHasher`によるハッシュとし、平文パスワード、外部IdP、ロール、MFA、リカバリーコードは保持しない。

初回設定と忘れた場合の再設定は、HTTP接続元とHostの両方がlocalhostまたはループバックIPである場合だけ許可する。通常のパスワード変更は認証後に現在のパスワードを再確認する。パスワード更新時は`SessionVersion`を増加させ、以前のCookieを無効化する。

ログイン、初期設定、再設定、言語変更以外の管理画面と、業務DB反映SQLのダウンロードAPIは認証必須とする。Blazor Server接続は匿名画面の描画にも使用するが、Routerの認可によって管理コンポーネントへ遷移させない。ログイン試行は接続元IPごとに1分間5回へ制限する。

表示言語はASP.NET CoreのカルチャーCookieに保存し、日本語と英語を選択可能にする。共通シェル、ダッシュボード、認証画面からリソース化し、業務固有画面は用語確定に合わせて拡張する。

## 理由

インターネット接続やActive Directoryを前提にせず、社内LAN内の小規模運用で無断変更を防ぐ最小境界が必要である。localhost限定復旧により、環境変数の恒久パスワードや配布・保管が必要なリカバリーコードを避ける。Cookie世代番号により、DBの認証レコードを削除せずにパスワード再設定と既存セッション無効化を行える。
