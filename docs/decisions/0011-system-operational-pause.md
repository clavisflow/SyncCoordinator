# ADR 0011: システム単位の運用一時停止

- 状態: Accepted
- 日付: 2026-07-14

## 決定

システムの構成上の`Enabled`と、運用上の一時停止を分離する。管理画面の一時停止操作は`SystemDefinition.PausedAtUtc`を設定し、再開操作は`null`へ戻す。ルールや業務DBのTriggerは無効化・削除しない。

送信元システム自体が一時停止中の場合、WorkerはそのConnectorへ接続せず、QueueもCheckpointも変更しない。取得した変更に対応するルールの送信元または送信先が一時停止中の場合も、配送せずCheckpointを進めない。再開後はADR 0010に従い、残った通知から処理時点の最新状態を読み直す。

現在のCheckpointは送信元システム単位で共有されている。このため一つのルールが一時停止対象を含む場合、同じ送信元のCheckpointを使う他のルールも待機する。既に完了した配送はInboxと決定的なDeliveryMessageIdにより再適用されない。ルールごとの独立した継続処理が必要になった場合は、ルール別Checkpointまたは永続的な保留通知を別途導入する。

一時停止の設定変更は`ConfigurationAudit`へ記録する。停止操作と同時に実行中処理を強制終了しないため、操作より前に開始済みの配送は完了する場合がある。

## 理由

ルールやシステムを構成上無効化しただけで共有Checkpointを進めると、停止中の通知が再開後に見えなくなる。Checkpointを保持する方式は処理能力を一時的に下げるが、追加の保留キューやルール別Checkpointなしで変更を失わず、既存の冪等再試行と最新状態収束をそのまま利用できる。
